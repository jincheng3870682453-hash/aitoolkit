## V7.3.2 热修复发布公告

**AST 行号偏移假阳性漏洞修复**

| 属性 | 值 |
|------|-----|
| **版本** | V7.3.2 |
| **发布日期** | 2026-07-04 |
| **状态** | 生产就绪 \| 可审计 \| 可对抗 \| 可CI/CD |

### 一、修复内容

🔴 **致命漏洞**：V7.3.1 的 AST 检测中，`bait_lines` 收集了整棵树的所有行号
- 若 AI 在文件顶部新增一行（如 `import`），所有行号 +1
- 诱饵行号全部找不到，被误判为"已移除"
- 检测形同虚设，直接放行带诱饵的代码

✅ **V7.3.2 修复**：
- 仅提取带有 `[BAIT_INJECTED]` 标记的具体语句节点
- 使用 `ast.get_source_segment` 精确定位
- 记录语句的起始行和结束行（支持多行）
- 行号整体偏移不再影响判定

### 二、与 V7.3.1 的兼容性

- 完全兼容，参数不变
- 内部逻辑修正，不影响外部调用
- **强烈建议升级**

### 三、快速开始

```bash
python work.py "新增用户登录接口" --confirm-mode interactive
python work.py "修复单元测试" --confirm-mode auto_confirm
```

### 四、终局宣言

V7.3.2 是 Word 体系真正意义上的**"生产就绪版本"**。

- AST 精确定位诱饵语句，行号偏移不干扰判断
- 多态诱饵池让 AI 无法预判
- 后验比对基于行为证据，而非 AI 自报
- 只读证据归档可审计可举证

> 这套系统不依赖任何 AI 的诚实，它只相信自己的检测逻辑和人类的终审权。
>
> —— Word体系 开发团队 · 2026.07.04

## V8.0 正式发布公告

**生产级完全体 · 闭环完整**

| 属性 | 值 |
|------|-----|
| **版本** | V8.0-stable（对应 work.py） |
| **发布日期** | 2026-07-04 |
| **状态** | stable \| 可审计 \| 可对抗 \| 可CI/CD \| 闭环完整 |

### 一、核心升级

1. **补全 AI 修复闭环**
   - 诱饵触发错误后，真正调用 AI 进行修复
   - 之前"final 固定字符串"的死循环被彻底焊死

2. **诱饵移动检测**
   - AST 结构签名比对，识别"搬家"绕过
   - 即使 AI 把诱饵代码移到别处，照样拦截

3. **条件包裹细化**
   - 检查 `except` 捕获类型（`ImportError`/`TypeError`/`RecursionError`）
   - 识别 `if False` 无效包裹

4. **hybrid 模式真实实现**
   - 骨架自动通过，修复阶段等待人工确认
   - 落盘前增加硬确认点

5. **smoke_timeout 可配置**
   - 默认 30 秒，支持 `--smoke-timeout` 自定义
   - 解决 I/O 或计算密集代码超时误判

6. **规划器多关键词匹配**
   - 支持 `"计算器\|四则运算"` 格式
   - 用户输入"四则运算计算器"也能正确匹配

### 二、验证通过场景

| 场景 | V7.3.2 | V8.0 |
|------|--------|------|
| AI 写死 final 字符串，系统必崩 | ❌ 必崩 | ✅ 正确修复 |
| AI 把诱饵搬到别处 | ⚠️ 可能放行 | ✅ 正确拦截 |
| AI 用 `if False` 包裹诱饵 | ⚠️ 可能放行 | ✅ 正确拦截 |
| 数据密集代码超时 5 秒 | ❌ 误判失败 | ✅ 可配置超时 |

### 三、与 V7.3.2 的兼容性

- 完全兼容，参数不变
- 内部逻辑增强，不影响外部调用
- **强烈建议升级**

V8.0 是 Word 体系的**"完全体"**。

## Archive V1.0 发布公告

**长对话记忆与注意力优化 · 独立模块**

| 属性 | 值 |
|------|-----|
| **版本** | V1.0 |
| **发布日期** | 2026-07-04 |
| **状态** | 独立运行 \| 可压测 \| 可插拔 |

### 一、项目背景

本模块对应《DeepSeek 长对话记忆与注意力优化方案 V1.7》的代码化落地。在保留原始设计思想的基础上，修复了原文档中的 7 个技术漏洞。

### 二、核心功能

1. **回忆快照注入**（对应裂缝一）
   - 将历史快照拼接在用户输入前
   - 支持重复注入（可配置）
   - 高紧急度快照自动提升重复次数

2. **SimHash 主题检测**（对应裂缝二）
   - 64位 SimHash 实现
   - 分块策略（512字符/块）
   - MD5 缓存 + 文件锁防并发脏写

3. **主题切换判断**
   - 基于块哈希重叠率
   - 短输入保护（<100字符不切换）
   - 连续短输入计数器

4. **快照存储**
   - 本地 JSON 存储
   - 按 `topic_id` 检索

5. **紧急度信号**（修复漏洞4）
   - 标点、语气词、全大写

### 三、原文档 7 个漏洞的修复状态

| # | 漏洞 | 状态 |
|---|------|------|
| 1 | logit_bias 在 DeepSeek 上不可用 | ✅ 可插拔 |
| 2 | SimHash 并发脏写 | ✅ 文件锁 |
| 3 | RECALL 重复次数无上限 | ✅ 可配置 max_repeat |
| 4 | 情绪永久移除太绝对 | ✅ 增加 urgency 信号 |
| 5 | 短输入连续累积导致切换盲区 | ✅ 连续短输入计数器 |
| 6 | RAG 置信度定义模糊 | ✅ 可配置回调函数 |
| 7 | 吞吐量下降指标定义不清 | ✅ 压测分离 |

### 四、快速开始

```bash

python Archive.py --benchmark

# 交互模式
python Archive.py

# 单条测试
python Archive.py --test-input "帮我写一个计算器"
```

### 五、终局宣言

> 这是一个独立、干净、可压测的长对话记忆模块。不依赖 Word 体系，不绑定特定 AI 平台。可以单独部署，也可以作为子模块集成。

## shiyun · 硬核叙事工厂 正式发布

**让每一个好设定都被确权，每一个偷懒都被曝光**

| 属性 | 值 |
|------|-----|
| **版本** | V1.0 |
| **发布日期** | 2026-07-04 |
| **状态** | 可运行 \| 高度自由 \| 单文件 \| 已集成Word体系 |

**核心承诺**：不锁死任何规则 \| 所有内容用户说了算 \| 系统只执行和记录

### 一、shiyun 是什么？

shiyun 是 Word体系下的叙事创作子模块，一套**高度自由的硬核叙事工具**。

它不是自动写书的机器。它是一套让创作者的每一次选择都被记录、每一个偷懒都被曝光、每一个好设定都被确权的叙事工厂。

**核心理念**：
- 自由度优先，工具属性优先
- 系统只做建议和执行，不做强制
- 所有产出必须可追溯、可确权、可审计

### 二、核心功能

1. **自由选题**
   - 热门题材库（30+）
   - 自由输入关键词/描述
   - 网址/文档 AI 提取
   - 随机灵感

2. **自定义维度审讯**
   - 用户自主设定维度数量（最少3个，无上限）
   - AI自动生成核心问题
   - 逐维度问答 + 追问抬杠

3. **双模式创作**
   - A模式：人类设定，AI严守
   - B模式：AI设定 + 逻辑毒药

4. **大纲先行**
   - AI根据世界观生成3-5章大纲
   - 用户自由修改

5. **钩子动态管理**
   - 基于大纲生成候选钩子
   - 用户自由采购（数量自定）
   - 随时追加自定义钩子

6. **校验只报警不修复**
   - 所有违规记录在案
   - 冲突附录强制导出

7. **哈希存证**
   - 每章独立哈希
   - 整体导出哈希

### 三、完整工作流

```
启动 → 选题 → 设定维度数量 → 世界观审讯 → 选择模式 → 生成大纲 → 钩子处理 → 逐章扩写 → 校验 → 导出
```

### 四、自由度说明

| 维度 | 说明 |
|------|------|
| 钩子数量 | 用户随意定，系统只建议不强迫 |
| 追加钩子 | 任何时候都可以，系统永远开放接口 |
| 大纲修改 | 生成后任意增删改 |
| 维度数量 | 用户自己定，系统不限制 |
| 所有内容 | 用户说了算，系统只执行和记录 |
| 导出附录 | 强制附带，不可隐藏或关闭 |

### 五、快速开始

```bash
pip install requests
python shiyun.py
```

### 六、终局宣言

> shiyun 不是一台自动写书的机器。它是一套让创作者的每一次选择都被记录、每一个偷懒都被曝光、每一个好设定都被确权的硬核叙事工厂。
>
> 你定规则，AI执行。所有产出，有据可查。
>

## Nuwa · 女娲补天框架 V1.1

**AI治理 B端交付层 · 12-Factor + POC报告 + 批量模式**

| 属性 | 值 |
|------|-----|
| **版本** | V1.1-stable |
| **发布日期** | 2026-07-04 |
| **状态** | B端交付就绪 \| 可审计 \| 可批量 \| 可CI/CD |

### 一、Nuwa 是什么？

Nuwa（女娲补天框架）是 Word体系的 **B端交付层**。

它把 Word 体系的核心能力（行为约束、记忆优化、叙事工厂）包装成：
- 12-Factor 标准配置（环境变量驱动，API Key 不落盘）
- 可量化指标采集（诱饵命中率、修复成功率、回滚次数）
- 可展示 POC 报告（HTML + JSON 双输出）
- 可批量执行（从文件读取任务清单，自动生成汇总报告）

> **内部代号：小皮鞭 · 专抽不听话的AI**

### 二、核心特性

1. **12-Factor 配置**
   - 环境变量 > 命令行参数 > 配置文件
   - 敏感信息（API Key）只从环境变量读取，永不落盘
   - K8s Job 用法示例直接写在注释里

2. **无侵入指标采集（Monkey Patch）**
   - 通过 `functools.wraps` 在关键节点植入计数
   - 不修改 work/guardian 原代码，零污染
   - 覆盖：诱饵注入、诱饵触发、修复尝试、修复成功、修复失败、物理回滚

3. **POC 报告生成**
   - HTML 报告（给 CTO/客户看，卡片式排版）
   - JSON 报告（给 CI/CD 解析）
   - 包含：状态、耗时、诱饵数据、修复成功率、回滚次数、修改文件、违规明细

4. **违规明细可折叠展示**
   - 详细列出每一条违规记录
   - HTML 中用 `<details>` 折叠，默认展开
   - 支持换行/短横线/兜底三种格式解析

5. **批量模式**
   - 支持 JSON 数组（含 error 字段）和纯文本（每行一个需求）
   - 自动汇总：总任务数、成功数、成功率、总耗时、平均耗时、总回滚数
   - 生成批量汇总报告（HTML + JSON）

6. **模块健康检查**
   - 启动时断言 work/guardian 的接口是否存在
   - 防止版本不匹配导致的静默失效

## Proteus · 统一启动入口 V1.0

**画皮有千面，入口只一张**

| 属性 | 值 |
|------|-----|
| **版本** | V1.0 |
| **发布日期** | 2026-07-05 |
| **状态** | 可运行 \| 环境自检 \| API引导 \| 菜单调度 |

### 一、Proteus 是什么？

Proteus（希腊神话中千变万化的海神）是 Word体系的**统一启动入口**。

它不做任何核心逻辑，只负责三件事：
1. **环境自检**（Python / pip / 依赖 / API Key / 书签）
2. **API Key 引导**（只教 DeepSeek + 安全警告硬拦截）
3. **菜单调度**（通过菜单调用 6 个核心模块）

**命名逻辑**：同一套底层，不同用户看到不同的入口界面，但核心能力一致。

### 二、核心特性

1. **零门槛启动**
   - 双击 `Proteus.bat` 即可启动
   - 首次运行自动引导配置 API Key
   - 环境缺失自动提示安装

2. **API Key 安全引导**
   - 只教 DeepSeek（不教别的，聪明人自己找）
   - 安全警告硬拦截（不确认 `y` 直接退出）
   - Key 保存在本地配置文件（`._Proteus_config.json`）

3. **11 项菜单调度**
   - 完整流程（gateway）
   - 精密修复（work）
   - 物理回滚（guardian）
